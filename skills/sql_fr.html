SQL

Le SQL, ou “Structured Query Language” ("Langage de Requêtes Structurées" en français) est un langage permettant de créer et de gérer des bases de données, étant décliné en de nombreuses versions. Parmi ces versions, j’ai eu particulièrement l’occasion de travailler sur MySQL, SQLite, et HFSQL, dont voici les spécifications :

  • MySQL est une version extrêmement versatile de SQL. Nous pouvons l’employer pour des applications Linux, MacOS, Windows, ou encore web. Elle est également très populaire, utilisée par un grand nombre de sites et d’applications. Nous pouvons en trouver deux types de licence : gratuite et payante ; bien que je n’aie travaillé que sur la licence gratuite.
  • SQLite est une version très légère de SQL. Elle est surtout employée pour les applications mobiles, où l’espace de stockage est beaucoup plus limité. Cette version est open source.
  • HFSQL est une version modifiée de SQL pour répondre aux besoins spécifiques d’un type de langage de programmation particulier, nommé le WLangage, mis au point par la société montpelliéraine PC Soft. Contrairement aux deux autres versions citées plus haut, il s’agit d’une version propriétaire appartenant à PC Soft.

C'est à travers ces versions que je me suis forgé une expérience du SQL, dont voici les principales :

  • C’est à travers MySQL que j’ai appris les bases du langage SQL, en le découvrant lors de ma première année d’études, afin de réaliser un site internet. Je le réemploierai deux ans plus tard, lors de ma troisième année d’études, afin de réaliser une application bien plus complète, du nom de Phytolearning, projet de création de formations pour une entreprise d'analyse scientifique.

  • Afin de réaliser une application Android, nommée Hermès, j’ai eu à employer SQLite, cette version étant particulièrement adaptée à cette technologie. À travers cette version, j’ai pu réaliser les tables nécessaires à la saisie de fiches personnelles, permettant aux utilisateurs de l'application de sauvegarder lesdites données. L'application ayant pour but d'empêcher l'inversion de cendres de personnes crémées, l'importance de la sauvegarde des données est ainsi soulignée, d'où l'intérêt de l'usage d'un système de stockage léger, simple et reconnu.

  • En ce qui concerne HFSQL, c’est une technologie avec laquelle je me suis particulièrement familiarisée lors de mon arrivée en entreprise, et en travaillant sur le projet Portail R&R, l’entreprise en question travaillant presque exclusivement avec des technologies « WLangage ». Cette version de SQL intègre une syntaxe différente, retravaillée, afin d'accomoder le WLangage. Son intérêt réside donc dans cette intercommunication, accélérant notre développement.

La réalisation de bases de données est un élément extrêmement récurrent lors de la création d'applications et de logiciels, une base de données étant l'élément permettant d'assurer la continuité du travail des utilisateurs de l'application. Et SQL fait partie des familles de langages spécialisés dans la réalisation de base de données les plus populaires.
Bien qu’utilisant régulièrement des solutions issues de SQL, car étant ma technologie de prédilection pour la réalisation de bases de données, la grande versatilité de SQL fait que, bien qu’en ayant utilisé plusieurs versions, je ne peux en revendiquer la maîtrise totale, bien au contraire.

Pour remédier à ce problème, j’envisage de continuer à me former à cette technologie, en essayant d’obtenir une certification, comme par exemple celle délivrée par W3School, et en mettant en place une veille concernant les dernières versions de ce langage rendues accessibles au public.
J'aimerai pouvoir, dans un avenir proche, ajouter au moins trois autres versions de SQL que je serai susceptible d'employer régulièrement, et avec le même niveau que celui dont je dispose aujourd'hui, présenté au sein de ces lignes.
